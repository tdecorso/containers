Detailed Description

Hash map implementing a generic key-value store.

Uses separate chaining (one list_t per bucket). Keys and values are stored inline in a flat [key | value] byte buffer inside each list node, so no per-entry heap allocations are needed beyond the node itself.

The table resizes automatically when count / bucket_count exceeds max_load_factor.
