containers.h — A generic data structures library in C

Introduction

containers.h is a type-agnostic data structures library written in C99. It provides the most common container types through a uniform API: every container operates on void* with caller-supplied element sizes, reports errors through an optional error_t output parameter, and copies data by value so the caller retains ownership of its memory.

This is primarily an instructional project. The goal is to produce clean, readable, well-documented C code rather than a production-grade, performance-optimised library.

Building

Requirements: CMake 3.31+, a C99-compatible compiler.

mkdir build && cd build
cmake ..
cmake --build .

This produces libcontainers.a (or containers.lib on Windows) in the build directory. To link the library into your own CMake project:

target_link_libraries(my_target PRIVATE containers)

Error handling

Every fallible function accepts a trailing error_t* err parameter. Passing NULL is always safe — errors are silently ignored. On failure the function populates err->code with one of the error_e values and err->msg with a human-readable description.

error_t err;
array_t* arr = array_create(sizeof(int), 16, &err);
if (!arr) {
fprintf(stderr, "error %d: %s\n", err.code, err.msg);
return 1;
}

Working with generic containers

All containers store copies of your data. You provide the element size at creation time; the library handles the rest:

typedef struct { float x, y; } vec2_t;
array_t* points = array_create(sizeof(vec2_t), 64, NULL);
vec2_t p = { 1.0f, 2.0f };
array_push_back(points, &p, NULL);
vec2_t* q = (vec2_t*)array_front(points, NULL);
printf("%.1f %.1f\n", q->x, q->y);
array_destroy(points);

N-ary trees

Trees are built by appending children and siblings to existing nodes. The root is created by passing NULL as the parent to tree_append_child. Nodes can also be inserted at arbitrary positions with tree_insert_before and relocated with tree_move. Each node tracks its direct child count in O(1) via tree_child_count, and the sibling chain is doubly-linked so tree_prev_sibling is also O(1).

tree_t* tree = tree_create(sizeof(int), NULL);
int v;
v = 1; tree_node_t* root = tree_append_child(tree, NULL, &v, NULL);
v = 2; tree_node_t* c1 = tree_append_child(tree, root, &v, NULL);
v = 3; tree_append_child(tree, root, &v, NULL);
v = 4; tree_append_child(tree, c1, &v, NULL);
v = 5; tree_append_child(tree, c1, &v, NULL);

Built-in pre-order and post-order walks accept a tree_walk_fn callback and a caller-supplied context pointer:

typedef struct { int sum; } sum_ctx_t;
tree_walk_result_t sum_cb(tree_node_t* node, size_t depth, void* userdata) {
sum_ctx_t* ctx = (sum_ctx_t*)userdata;
ctx->sum += *(int*)tree_node_data(node, NULL);
return TREE_WALK_CONTINUE;
}

Notes and limitations

  • Not thread-safe. External synchronisation is required for concurrent access to any container.
  • No iterators for arrays. Use index-based access via array_at, array_front, and array_back.
  • No cycle detection in tree_move. Moving a node to be a descendant of itself is undefined behaviour; the caller is responsible for preventing it.
  • Struct keys with padding must be zero-initialised with memset before use as hash map keys, or hashing will produce inconsistent results across equal structs.
  • This project is in early development. Interfaces may evolve as additional containers are introduced.
